# Definition of Done

A task is done only when:

- The implementation satisfies the Linear issue acceptance criteria
- Scope matches the issue and out-of-scope items were not included
- Tests pass
- Lint and typecheck pass
- Permissions and visibility rules are preserved
- UI includes empty/loading/error states where relevant
- PR is linked to the Linear issue
- PR description includes summary, test notes, and follow-ups
- Screenshots are attached for UI changes
